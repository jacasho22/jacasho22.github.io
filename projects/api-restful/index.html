Primeros Pasos

Para comenzar a utilizar nuestra API RESTful, sigue estos sencillos pasos:

1. Registro y Obtención de API Key

Regístrate en nuestra plataforma para obtener tu API Key personal que necesitarás para autenticar tus solicitudes.

2. Instalación

Puedes interactuar con nuestra API directamente mediante solicitudes HTTP o utilizando nuestro SDK oficial:

npm install api-restful-client
yarn add api-restful-client
<script src="https://cdn.ejemplo.com/api-restful-client.min.js"></script>

3. Configuración

Configura el cliente con tu API Key:

// Inicializar el cliente
const apiClient = new ApiRestfulClient({
  apiKey: 'TU_API_KEY',
  baseUrl: 'https://api.ejemplo.com/v1'
});

4. Realizar tu primera solicitud

Ahora puedes comenzar a realizar solicitudes a la API:

// Obtener todos los usuarios
apiClient.users.getAll()
  .then(response => {
    console.log('Usuarios:', response.data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Autenticación

Nuestra API utiliza autenticación basada en JWT (JSON Web Tokens) para proteger los recursos y verificar la identidad de los usuarios.

Obtener un Token

Para obtener un token de acceso, debes realizar una solicitud POST al endpoint de login:

// Iniciar sesión y obtener token
apiClient.auth.login({
  username: 'usuario',
  password: 'contraseña'
})
.then(response => {
  // Guardar el token para futuras solicitudes
  const token = response.data.token;
  console.log('Token:', token);
})
.catch(error => {
  console.error('Error de autenticación:', error);
});

Usar el Token en Solicitudes

Una vez que tienes un token, puedes incluirlo en tus solicitudes de dos formas:

1. Configuración Global

// Configurar el token para todas las solicitudes
apiClient.setAuthToken(token);

2. Por Solicitud Individual

// Incluir el token en una solicitud específica
apiClient.products.create(productData, { token: token });

Endpoints Principales

Nuestra API ofrece los siguientes grupos de endpoints:

Usuarios

  • GET /api/v1/users - Obtener todos los usuarios
  • GET /api/v1/users/:id - Obtener un usuario por ID
  • POST /api/v1/users - Crear un nuevo usuario
  • PUT /api/v1/users/:id - Actualizar un usuario existente
  • DELETE /api/v1/users/:id - Eliminar un usuario

Productos

  • GET /api/v1/products - Obtener todos los productos
  • GET /api/v1/products/:id - Obtener un producto por ID
  • POST /api/v1/products - Crear un nuevo producto
  • PUT /api/v1/products/:id - Actualizar un producto existente
  • DELETE /api/v1/products/:id - Eliminar un producto
  • POST /api/v1/products/:id/rate - Calificar un producto

Autenticación

  • POST /api/v1/auth/login - Iniciar sesión
  • POST /api/v1/auth/register - Registrar nuevo usuario
  • GET /api/v1/auth/verify - Verificar token
  • POST /api/v1/auth/logout - Cerrar sesión

Para obtener información detallada sobre cada endpoint, parámetros y respuestas, consulta nuestra documentación completa.

Manejo de Errores

Nuestra API utiliza códigos de estado HTTP estándar para indicar el éxito o fracaso de una solicitud. Además, todas las respuestas de error incluyen un objeto JSON con detalles adicionales.

Estructura de Respuesta de Error

{
  "status": 400,
  "error": "Bad Request",
  "message": "Descripción detallada del error",
  "errors": [
    {
      "field": "email",
      "message": "El formato del email no es válido"
    }
  ]
}

Códigos de Estado Comunes

  • 200 OK - La solicitud se completó con éxito
  • 201 Created - El recurso se creó correctamente
  • 400 Bad Request - La solicitud contiene datos inválidos
  • 401 Unauthorized - Autenticación requerida o inválida
  • 403 Forbidden - No tienes permisos para acceder al recurso
  • 404 Not Found - El recurso solicitado no existe
  • 422 Unprocessable Entity - Validación fallida
  • 429 Too Many Requests - Has excedido el límite de solicitudes
  • 500 Internal Server Error - Error interno del servidor

Manejo de Errores en el Cliente

apiClient.users.getById(999)
  .then(response => {
    console.log('Usuario:', response.data);
  })
  .catch(error => {
    if (error.status === 404) {
      console.error('Usuario no encontrado');
    } else if (error.status === 401) {
      console.error('Sesión expirada, inicia sesión nuevamente');
      // Redirigir al login
    } else {
      console.error('Error:', error.message);
    }
  });
